## 七、问题根因分析

### 7.1 为什么会产生这些问题？

**根本原因**: 提示词设计缺陷

| 原提示词问题 | 导致的后果 |
|--------------|------------|
| "可以虚构人物名字，但故事要基于真实事件" | AI理解为"允许虚构"，忽略"基于真实"的限制 |
| 缺乏数据来源要求 | AI编造具体数字 |
| 缺乏事实核查机制 | 无法验证的历史数据被直接使用 |
| 案例示例本身为虚构 | AI模仿生成更多虚构案例 |

### 7.2 提示词设计错误模式

```
❌ 错误模式: "可以虚构X，但要满足Y"
   → AI选择性忽略Y，只执行X

❌ 错误模式: 提供虚构案例作为"好示例"
   → AI模仿生成类似虚构内容

❌ 错误模式: 缺乏明确的数据来源要求
   → AI编造数字以满足"具体细节"要求
```

## 八、优化建议

### 8.1 提示词修改方案

#### 修改1: 案例规范 (已修正)
**原提示词**:
```
- 可以虚构人物名字，但故事要基于真实事件
```

**修正后**:
```
**核心原则: 所有案例必须基于真实事件，禁止虚构人物和数据。**

### 允许的案例来源
- 新闻报道中的真实事件（需标注"据报道"）
- 官方发布的数据和典型案例
- 你亲身经历或亲眼所见的事
- 历史文献记载的真实事件

### 禁止的行为
- ❌ 禁止虚构"老张""小林"等人物及其故事
- ❌ 禁止编造具体数字（如"月薪8500""增长67%"）
- ❌ 禁止编造无法验证的细节
```

#### 修改2: 数据规范 (已修正)
**新增**:
```
## 数据使用规范（重要）

### 数据来源要求
**所有数据必须有明确来源，禁止编造。**

### 允许使用的数据
- ✅ 官方发布的数据（政府、权威机构）
- ✅ 知名研究机构发布的报告（需标注来源）
- ✅ 可查证的统计数据
- ✅ 你亲自核实过的数据

### 禁止的行为
- ❌ 禁止编造具体数字（如"增长67%""月薪8500"）
- ❌ 禁止使用无法验证的"调研数据"
- ❌ 禁止将估算值当作精确数据

### 不确定时的处理
- 使用定性描述代替具体数字："快速增长"代替"增长67%"
- 标注数据来源："根据人社部数据...""据报道..."
- 承认不确定性："具体数据不详，但趋势明显"
```

#### 修改3: 质量检查清单 (已新增)
```
## 自检清单（生成后必须检查）

- [ ] 所有案例是否基于真实事件？（禁止虚构人物）
- [ ] 所有数据是否有明确来源？（禁止编造数字）
- [ ] 不确定的信息是否标注了"据报道"或"据估算"？
```

### 8.2 文章重写建议

若需重写该文章，建议：

1. **替换虚构案例**
   - ❌ "老张" → ✅ "据报道，某招聘平台数据显示..."
   - ❌ "设计师小林" → ✅ "根据某广告公司案例..."

2. **标注数据来源**
   - ❌ "增长了67%" → ✅ "根据人社部发布的数据..."
   - ❌ "72%被动离职率" → ✅ "某调研机构数据显示..."（或删除）

3. **增加不确定性标注**
   - "具体数据不详，但从趋势来看..."

## 十、总结

### 核心发现
1. **虚构内容是最大问题**: 5个核心案例全部为虚构
2. **数据来源缺失**: 关键数据无出处，无法验证
3. **提示词设计缺陷**: "可以虚构"的表述被AI选择性执行

### 关键教训
- **禁止类指令必须明确**: "禁止虚构"比"可以虚构，但要..."更有效
- **示例必须是真实的**: 提供虚构案例作为"好示例"会导致模仿
- **数据来源必须强制**: 没有来源要求，AI会编造数字
