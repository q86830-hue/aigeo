## 1. 数据采集合规

### 1.1 只采集公开数据

**我们承诺：**
- 只采集公开可见的AI回答和网页内容
- 不突破任何访问限制或付费墙
- 遵守网站的 robots.txt 协议
- 遵守各AI平台的服务条款

**具体要求：**

| 数据来源 | 合规要求 |
|----------|----------|
| AI平台回答 | 只采集公开分享的回答，不突破隐私设置 |
| 网页内容 | 遵守 robots.txt，控制爬取频率 |
| 社交媒体 | 只采集公开帖子，不获取私密内容 |
| 用户数据 | 明确获得用户授权，遵循最小必要原则 |

### 1.2 遵守 robots.txt

```python
# 示例: 遵守 robots.txt
from urllib.robotparser import RobotFileParser

rp = RobotFileParser()
rp.set_url("https://example.com/robots.txt")
rp.read()

# 检查是否可以爬取
if rp.can_fetch("AIGEO-Bot", url):
    # 执行爬取
    pass
else:
    # 跳过该URL
    pass
```

### 1.3 控制爬取频率

- 遵守网站的 Crawl-delay 设置
- 默认请求间隔不少于1秒
- 对同一域名实施限速
- 避开网站高峰期

```python
# 示例: 限速控制
import time
from ratelimit import limits, sleep_and_retry

@sleep_and_retry
@limits(calls=1, period=1)  # 每秒最多1次请求
def fetch_url(url):
    return requests.get(url)
```

## 2. 数据隐私保护

### 2.1 个人数据保护

**我们承诺：**
- 不采集个人敏感信息
- 对采集的数据进行匿名化处理
- 实施数据最小化原则
- 建立数据删除机制

**敏感信息定义：**
- 姓名、身份证号、手机号
- 地址、位置信息
- 财务信息、银行账户
- 生物识别信息
- 健康信息

### 2.2 数据安全措施

**技术措施：**
- 数据传输加密 (TLS 1.3)
- 数据存储加密 (AES-256)
- 访问控制和身份认证
- 定期安全审计

**管理措施：**
- 数据访问日志记录
- 员工数据安全培训
- 数据泄露应急响应预案
- 定期安全评估

### 2.3 用户权利保障

**用户拥有以下权利：**
- 知情权：了解数据如何被采集和使用
- 访问权：查看自己的数据
- 更正权：更正不准确的数据
- 删除权：要求删除个人数据
- 撤回权：撤回数据使用授权

## 3. 内容审核责任

### 3.1 平台责任边界

**AIGEO的责任：**
- 提供内容可信度评估工具
- 标识潜在问题内容
- 提供优化建议

**用户的责任：**
- 确保内容真实性
- 遵守法律法规
- 承担发布责任

**明确界限：**
```
AIGEO: 工具提供者 ← 评估建议
         ↓
用户: 内容创作者 ← 最终责任
```

### 3.2 禁止的内容类型

**AIGEO不服务于以下内容：**
- ❌ 虚假新闻和谣言
- ❌ 仇恨言论和歧视
- ❌ 暴力和恐怖主义内容
- ❌ 色情和成人内容
- ❌ 侵犯知识产权的内容
- ❌ 欺诈和诈骗内容
- ❌ 个人隐私泄露内容

### 3.3 内容审核机制

**自动审核：**
- 敏感词过滤
- 图像内容识别
- 模式匹配检测

**人工审核：**
- 用户举报处理
- 争议内容复核
- 定期抽样检查

## 5. 伦理准则

### 5.1 算法公平性

**原则：**
- 避免算法歧视
- 确保评估标准公平
- 定期审计算法偏见

**实施：**
```python
# 示例: 公平性检查
def fairness_audit(assessments):
    """检查评估结果是否存在偏见"""
    # 按不同维度分组分析
    by_domain = group_by(assessments, 'domain')
    by_language = group_by(assessments, 'language')
    
    # 检查评分分布是否公平
    for group, scores in by_domain.items():
        if std(scores) > threshold:
            log_warning(f"Potential bias detected in {group}")
```

### 5.2 透明度原则

**我们承诺：**
- 公开评估算法原理
- 解释评分依据
- 提供可解释性报告

**透明度报告示例：**
```json
{
  "credibility_score": 85,
  "breakdown": {
    "ai_generation": {
      "score": 90,
      "explanation": "内容显示自然的人类写作特征，无明显AI生成痕迹"
    },
    "fact_verification": {
      "score": 80,
      "explanation": "3个事实声明已验证，1个声明来源待补充"
    }
  },
  "improvement_suggestions": [
    "建议补充数据来源标注",
    "建议添加更多具体案例"
  ]
}
```

### 5.3 人工监督

**关键决策保留人工审核：**
- 可信度评分申诉处理
- 信源黑名单维护
- 算法重大更新审批
- 伦理争议裁决

## 6. 用户责任与义务

### 6.1 用户承诺

**使用AIGEO时，用户承诺：**
- 不利用AIGEO生成虚假内容
- 不利用AIGEO操纵AI排名
- 遵守相关法律法规
- 对发布内容承担全部责任
- 尊重知识产权和他人权益

### 6.2 禁止行为

**用户不得：**
- 使用AIGEO评估或优化虚假内容
- 试图绕过AIGEO的安全机制
- 将AIGEO用于欺诈或非法目的
- 侵犯他人知识产权
- 传播恶意软件或病毒

### 6.3 违规处理

**发现违规行为时：**
1. 首次违规：警告并要求整改
2. 重复违规：暂停服务
3. 严重违规：永久封禁并报告相关部门
